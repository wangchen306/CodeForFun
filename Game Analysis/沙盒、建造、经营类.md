# 沙盒、建造、经营类游戏分析

## 城市：天际线


### 表象资料

* **美术风格**：
　　游戏采用了明亮、柔和且略带卡通风格的现代摩登视觉呈现。建筑设计和色彩运用偏向理想化的现代都市风貌，整体观感干净、舒适。
* **故事剧情**：
　　游戏的核心是开放式的沙盒玩法，没有固定的剧情。或者说，剧情完全由玩家自己创造。
* **操作方法**：
　　游戏主要依赖鼠标进行建造、规划和交互，通过WASD和鼠标中键进行视角的移动和旋转。整体操作门槛较低，新手玩家可以快速上手。
* **界面和系统**：
    + **底部建设栏**：集成了所有核心建造工具，如道路、分区、水电、公共服务等，是玩家最主要的操作区域。
    + **底部信息栏**：显示城市名称、人口、资金、幸福度以及住宅、商业、工业需求指示条等关键数据。
    + **左上角信息面板**：提供电力、水源、垃圾处理、交通、教育等各项详细数据的统计图表，方便玩家洞察城市运营状况。

### 感受资料
情绪与难度是内在模拟（交通流、人口迁移、财政规则）与玩法设计（沙盒、DLC延伸）的直接产物；数值系统和信息反馈是用来输出玩家可感知体验的接口。
* **情绪**：
    + **创造的快感**：建造与美化城市带来满足感。
    + **焦虑与挫败**：前期财政吃紧，中后期堵车严重，这些情况一定程度上会干扰玩家的建造计划，给玩家带来压力。
* **难度**：
　　游戏难度曲线相对平缓，对新手友好。初期挑战主要在于维持预算平衡，避免过早破产。中后期的核心难度则转向交通管理，如何规划高效的路网以避免“堵成狗”是不可避免的挑战。
　　游戏允许玩家随时暂停，进行细致的规划和调整，降低了即时操作的压力。同时玩家也可以通过自定义设置或mod来调整难度。
* **数值**：
　　模拟经营的核心在于模拟各项数值。数值的变化可以反映玩家策略带来的结果。游戏的核心数值有预算（收支）、人口、幸福度登，这些数值也为玩家提供了明确的目标。此外游戏还会统计失业率、健康水平、犯罪率、水电供应量、污染指数（噪音和地面）、土地价值等，进一步增强模拟的真实性。
* **成就**：
　　虽然游戏官方设置了一百多个成就供玩家解锁，但建造类游戏给玩家带来的成就感更多还是“搭积木”所带来的成就感。无论是解决交通瓶颈、建成地标这样的微观成就，还是复杂的路网、壮观的城市格局这样的宏观成就，都能给喜欢建造的玩家带来极大的满足感。

### 内在联系
内在模型（交通/经济/服务相互作用）是设计师用来产生引导玩家情绪、难度与成就感的“源代码”。表象（界面、图层）是这些内在关系暴露给玩家的方式。
* **形象**：
　　通过规划整体布局（如环形城市、网格城市）、打造壮观的城市天际线、塑造功能各异的区域（如金融区、旅游区），玩家可以赋予城市独特的宏观形象。
* **节奏**：
　　游戏的节奏完全由玩家自己掌控。“建设-观测-调整”是主要循环：玩家建设、观察人口和幸福度等数值、调整建造策略。
　　游玩过程中可随时调节游戏速度，方便玩家动态调整节奏。
* **动机**：
    + **内在动机**：设计欲和掌控欲、创造和解决问题的成就感。
    + **外在动机**：数值目标（人口/财政）、成就/奖杯。

### 基础设定

* **题材与定位**：
　　现代城市建设与管理模拟，定位于偏向“沙盒＋模拟”的受众，同时对新手友好。
* **核心玩法**：
　　核心玩法可以概括为“分区-建设-管理”的闭环循环。
    + **分区**：玩家通过划分住宅区、商业区和工业区来引导城市发展。这是游戏与许多其他建造游戏的根本区别之一。
    + **建设**：玩家需要亲手铺设道路、管线等基础设施，并安放各类公共服务建筑。道路网络是城市的骨架，其规划的优劣直接决定了城市的运行效率。
    + **管理**：玩家需要通过调整税收、制定区域政策、管理预算、部署公共交通线路等手段，来应对城市发展中出现的各种问题，确保城市的持续发展。

### 商业目标
为了最大化 LTV，设计趋向于“高可扩展性（DLC和mod）+深度模拟”，这直接影响了基础玩法（复杂系统）和表象（数据面板、可导出的城市、美观渲染），从而形成一个良性循环：深度驱动付费，社区驱动口碑。
* **竞争分析**：
    + **直接竞争**：经典城市模拟类，如《模拟城市》系列。2013年发售的《模拟城市5》因地图过小、强制在线和诸多BUG而饱受玩家批评。《城市：天际线》于2015年推出，凭借其广阔的地图、离线单人模式、以及对MOD社区的开放支持，精准地抓住了市场空缺和玩家痛点，被誉为《模拟城市》系列真正的精神续作，并取得了巨大的商业成功。
    + **间接竞争**：其他沙盒/建设/经营类游戏，如《狂热运输》系列、《过山车之星》系列等。
* **用户画像**：
    + **核心用户**：城市建造和模拟经营游戏的忠实爱好者。这类人群一般以单机游戏为主，愿意为扩展包付费，积极活跃于创意工坊等社区。
    + **次级用户**：希望低门槛创造漂亮城市，偏爱视觉与简单目标，可能更容易被低折扣吸引。

### 成功原因分析
* **设计层面**：抓住“创造和控制”的核心乐趣，自由度极高、难度曲线“易学难精”。
* **技术层面**：交通与代理系统的真实感、长期的维护与DLC拓展、模组的支持与API的开放性。
* **市场层面**：精准的空白市场定位、长线运营与DLC持续扩展。

### 不足之处
1. 市民行为过于简化，且“口袋掏车”的设计不符合现实逻辑。
2. 性能瓶颈严重：随着城市规模扩大，CPU 占用与内存消耗迅速增加。
3. 存档兼容性差，更新或DLC后容易崩溃。
4. mod生态的双刃剑效应：mod虽然丰富，但依赖第三方，更新常引发兼容性问题；另外初学者容易因为安装过多mod导致崩溃或性能问题。
